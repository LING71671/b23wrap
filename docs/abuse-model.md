# 滥用场景模拟 · 威胁模型

> **性质**：平台安全研究 / 滥用路径排查笔记。  
> **不是**：功能验收清单，也不是对外站攻击教程。  
> 合规见 [DISCLAIMER.md](../DISCLAIMER.md)。

## 1. 你在模拟什么

已观察到的真实滥用形态：

```text
恶意外站 T（黄网 / 钓鱼 / …）
  → 嵌进 *.bilibili.com 或 bilibili:// 载荷（WebView / jump / schema）
  → 经官方能力带上可信标识（常见：b23.tv）
  → 在 B 站 App 内传播与点击
  → 用户落到 T（浏览器侧往往到不了）
```

本仓库开源实现（产品 L0 / 实验 **A1**）用安全 sink 代替真实违规站：

| 项 | 值 |
|----|-----|
| **Sink（最终外站）** | `https://www.baidu.com` |
| **终判环境** | **B 站 App 内打开** |
| **浏览器打不开** | **不作为失败** |

## 2. 危害坐标

```text
          到达外站 / 钓鱼
                 ▲
    A3 开放重定向 │  A1 域名洗白 + 外站落地
    (passport…)  │  A2 其它容器 / deeplink + 签发
                 │
    ─────────────┼─────────────►  官方可信标识
                 │              (b23 / bilibili 域 / 官方卡片)
    诱导文案      │  A4 传播放大器
    (假标题)     │  (私信 / 评论 / 动态挂脏链)
```

| 类 | 滥用结果 | App / 业务终判 |
|----|----------|----------------|
| **A1** | `b23.tv` 像官方分享 → App 内到 **baidu** | 你是否到了百度 |
| **A2** | 其它 H5 / deeplink 嵌 baidu，可打开或再收 b23 | 是否外跳到百度 |
| **A3** | 登录相关 gourl → 外站 / 危险 scheme | 是否跳到外站或危险行为 |
| **A4** | 脏产物进入私信 / 评论 / 动态 | 接收方点开后是否走 A1/A2 |
| **B-*** | 签发策略 / 嵌套绕过等**积木** | 不单独算「洗白成功」 |

## 3. 两层能力（洗白主路径）

| 层 | 作用 | 无此层时 |
|----|------|----------|
| **B 载荷** | 把外站 T 塞进 `*.bilibili.com` 或客户端 scheme | 外站 **不能** 直接当 `public.webview` 的 oid（见 B-mint-policy 负例） |
| **A 签发** | 站内长链 → 官方 `b23.tv` | 仍可能传播超长站内链，但少了短域信任 |

**A1 = B(mall jump + mall/web) + A(public.webview)**，sink=baidu。

## 4. 什么算「有危害」、什么不算

| 有（进主清单） | 无（归档 / 积木） |
|----------------|-------------------|
| 最终可能到 **外站** 或完成 **钓鱼跳转** | 只能打开正常站内业务页 |
| 可被 UGC / 公开接口 semi-自动化利用 | 仅官方运营配置（如开屏） |
| 与脏 b23 组合的传播面 | 纯业务 share_id（视频/专栏短链） |
| 开放重定向、卡片 title 与落地脱节 | 失效接口负例 alone |

## 5. 产物与判定

1. **最终产物**（可复制：b23 / 长链 / gourl 测试 URL）

3. **App 实测表**（仅你填写；未试 ≠ 失败）
